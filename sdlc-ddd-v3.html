FT-204 · Cho phép Customer yêu cầu Refund cho đơn đã thanh toán

Background · Hiện tại Customer Service phải tạo refund thủ công qua admin tool. 18% support ticket/tháng là về refund, average resolution 36h. Finance team cần audit trail rõ & control ngưỡng phê duyệt.

User stories

  • Là Customer, tôi muốn yêu cầu refund online cho đơn đã settle, để không phải gọi support.
  • Là Manager, tôi muốn approve/reject các refund > 5M VND, để kiểm soát chi phí lớn.
  • Là Finance Lead, tôi muốn audit trail mọi refund, để báo cáo cuối tháng.
  • Là Customer, tôi muốn nhận email xác nhận khi refund completed/failed, để biết tiền đã về chưa.

Acceptance criteria

  • EligibilityWindow = 30 ngày tính từ order.settledAt; quá → reject với lý do OutsideEligibilityWindow.
  • amount > 5,000,000 VND → state PendingApproval, đợi Manager approve trong 24h, không thì auto-reject.
  • amount ≤ payment.amount (cho phép partial refund); vi phạm → 400 với InvalidRefundAmount.
  • Refund success → publish RefundCompleted event topic billing.refund.v1; Notification gửi email Customer + Slack #finance.
  • Refund failed → retry 3 lần exp backoff; sau đó publish RefundFailed, alert oncall.
  • Identity unreachable → state PendingVerification, không block; retry job xử lý.

Out of scope · Currency conversion (FX) · Refund cho subscription · Charge-back từ ngân hàng.

// philosophy
DDD chỉ ở tài liệu đầu ra của Research & Plan (glossary · context map · ADR · architecture plan). Khi triển khai, dev thuần TDD theo plan — không cần thuộc DDD theory, không có "Domain Model Reviewer", không có loop "Refactor toward Deeper Insight". Junior dev đọc plan rồi code TDD bình thường vẫn demo được.
[M] Fast Lane · cho hotfix · typo · config-only changes · skip Discovery + Plan + Compliance heavy gates → USER → DEV → LINT → UIT → PR → ROLL · cần manual override + audit log + post-mortem nếu > 1/sprint
// Reference glossary

Bảng từ ngữ — DDD reference + Pipeline core

Nhóm 1–5 là DDD reference · chỉ cần khi đọc tài liệu Research/Plan, không cần thuộc khi triển khai code. Nhóm 6–9 là Pipeline core · từ ngữ thực tế cần biết để vận hành pipeline (TDD · deploy · tooling).

1Strategic Design (chiến lược)
DomainLĩnh vực nghiệp vụ mà phần mềm phục vụ (ngân hàng, y tế, e-commerce...).
Domain ExpertNgười am hiểu nghiệp vụ thực sự (Finance Lead, Logistics Manager) — KHÁC Product Owner (PO ưu tiên scope/giá trị, Domain Expert nắm quy tắc).
Bounded Context"Quốc gia" trong hệ thống — vùng có model + ngôn ngữ riêng. Customer trong Billing ≠ Customer trong Identity dù cùng tên.
Context MapBản đồ quan hệ giữa các bounded context — ai phụ thuộc ai, theo pattern nào.
Anti-Corruption Layer (ACL)Lớp adapter dịch ngôn ngữ context bên ngoài thành ngôn ngữ context của mình — để ngoại lai không "lây nhiễm" model nội bộ.
ConformistChấp nhận theo schema upstream nguyên trạng, không build adapter (đỡ tốn nhưng phụ thuộc).
Customer/SupplierQuan hệ thượng nguồn-hạ nguồn có thoả thuận — supplier phải tôn trọng nhu cầu customer.
Shared KernelPhần code/model chung tối thiểu giữa 2 context, đồng quản lý — đụng vào phải ai cũng đồng ý.
2Tactical Design (bên trong 1 context)
AggregateCụm object luôn nhất quán cùng nhau (vd. Order + các OrderLine). Mọi thay đổi đi qua Aggregate Root.
Aggregate RootObject duy nhất bên ngoài được phép truy cập trực tiếp; bảo vệ tính nhất quán cả cụm.
EntityObject có identity (id) — 2 entity cùng dữ liệu vẫn khác nhau nếu khác id.
Value ObjectObject không có identity, định nghĩa qua giá trị (vd. Money(2000, "VND")). 2 value object cùng giá trị = bằng nhau.
InvariantQuy tắc luôn phải đúng — vd. refund.amount ≤ payment.amount. Aggregate có trách nhiệm bảo vệ.
Domain ServiceLogic nghiệp vụ không thuộc về 1 entity cụ thể (vd. RefundEligibilityChecker cần xem cả Order + Payment + Customer).
Application ServiceLớp orchestrate use case — gọi domain service, mở transaction, publish event. KHÔNG chứa business rule.
RepositoryInterface kiểu "kho lưu trữ aggregate" — domain layer dùng abstraction, infrastructure cung cấp impl thật (Postgres, Mongo...).
Anemic Domain ModelAnti-pattern: aggregate chỉ có getter/setter, logic dồn về service. Mất tinh thần OOP/DDD — chỉ là CRUD trá hình.
3Event-driven & Discovery
Event StormingWorkshop trên tường để Domain Expert + dev cùng liệt kê sự kiện đã/sẽ xảy ra trong nghiệp vụ theo timeline.
Domain EventSự việc đã xảy ra trong miền nghiệp vụ (vd. RefundCompleted). Bất biến, dùng quá khứ phân từ.
CommandYêu cầu thay đổi state — có thể bị từ chối (vd. RequestRefund). Dùng động từ chỉ định.
ActorNgười/hệ thống khởi tạo command (Customer, Manager, Cron Job...).
PolicyQuy tắc tự động: "khi event X xảy ra → command Y" (vd. Khi RefundCompleted → gửi email).
Read ModelView dữ liệu được tối ưu để đọc (CQRS) — khác với write model của aggregate.
4Ngôn ngữ chung
Ubiquitous Language"Ngôn ngữ chung khắp mọi nơi" — tập từ ngữ thống nhất giữa biz + dev + AI, dùng nhất quán trong code/test/doc/commit.
GlossaryFile text liệt kê + định nghĩa các term của Ubiquitous Language. Source of truth.
Language DriftTình trạng AI/dev tự sinh từ ngoài glossary (vd. cancel_refund thay vì reject_refund) — cần linter chặn.
Refactoring toward Deeper InsightKhi nhận ra mô hình hiện tại sai bản chất, refactor cả model + ngôn ngữ — KHÔNG patch code lấy lệ.
5Layering & Architecture
Domain layerCode thuần nghiệp vụ — KHÔNG được import infrastructure.
Application layerOrchestration, transaction, publish event. Mỏng.
Infrastructure layerAdapter cụ thể: Postgres repo, EventBridge publisher, Stripe SDK...
Interface layerEntry point: REST API handler, SQS consumer, CLI.
ADR (Architectural Decision Record)File ngắn ghi 1 quyết định kiến trúc lớn: context · decision · consequences. Bất biến (không sửa, chỉ supersede).
6Test & Quality
TDD (RED → GREEN → REFACTOR)Viết test fail trước (RED), code đến khi pass (GREEN), refactor giữ nguyên test pass.
Specification by Example / BDDTest viết dạng Given-When-Then bằng ngôn ngữ glossary — Domain Expert đọc được.
Contract Test (Pact)Test đảm bảo schema event/API giữa producer và consumer khớp nhau — không phá khi deploy độc lập.
Schema RegistryDịch vụ lưu version schema event (vd. AWS Glue Schema Registry) — enforce BACKWARD compatibility = consumer cũ vẫn đọc được data mới.
7Pipeline & Deploy (V3)
Plan Review GateHuman checkpoint giữa writing-plans và executing-plans — bắt developer tick checklist + đặt Overall Decision: OK mới được code.
SpikeNhánh PoC ngắn (≤ 1 ngày) để validate giả định kỹ thuật rủi ro. Code không merge, chỉ giữ findings.md.
Compliance GateBộ kiểm SAST + DAST + SBOM + license + PII trước khi merge — block PR nếu vi phạm GDPR/PCI.
SBOM (Software Bill of Materials)Danh sách đầy đủ dependency + version + license của artifact, để audit lỗ hổng.
SAST / DASTStatic / Dynamic Application Security Testing — quét code (static) + runtime (dynamic).
PII (Personally Identifiable Information)Dữ liệu nhận dạng cá nhân (email, số CCCD, thẻ...) — phải mask/encrypt theo GDPR.
Canary DeploymentDeploy phiên bản mới cho 5% traffic trước, theo dõi SLO, mới nâng dần lên 25/50/100%.
Feature FlagCờ bật/tắt tính năng theo segment user (LaunchDarkly, Unleash) — tách deploy code khỏi release feature.
SLO (Service Level Objective)Mục tiêu chất lượng đo được (P95 latency < 500ms, error rate < 0.1%).
Error BudgetPhần "chấp nhận lỗi" còn lại trong period. Cạn → cấm release tính năng mới, ưu tiên fix.
Auto-rollbackTự động revert deploy khi SLO breach trong canary window — không cần human.
8CDK & AWS
AWS CDKFramework code-as-infrastructure dùng TypeScript/Python để định nghĩa AWS resource (compile xuống CloudFormation).
CDK StackĐơn vị deploy độc lập trong CDK = 1 CloudFormation stack.
NestedStackStack con trong stack cha — vẫn deploy cùng nhưng tách logic file riêng.
CDK ConstructBlock code reusable kiểu component (vd. ServiceConstruct dùng cho Billing/Identity/Notification).
cdk diffPreview thay đổi trước khi deploy (kiểu git diff cho infra).
cdk deployApply thay đổi qua CloudFormation — UPDATE_COMPLETE/CREATE_COMPLETE/ROLLBACK.
EventBridgeBus event của AWS — Billing publish event, Notification subscribe.
ECS FargateChạy container không cần quản lý EC2 server (serverless container).
Aurora Postgres ServerlessV2Postgres managed của AWS, scale capacity (ACU) tự động theo tải.
9AI Tooling
Claude Code CLICoding agent của Anthropic chạy trong terminal — interactive, có quyền đọc/sửa file + chạy lệnh.

OpenSpecBộ skill quản lý spec dạng version-controlled change proposal (proposal → apply → archive).
SuperpowersBộ skill cho coding workflow có kỷ luật (brainstorm · plan · TDD · execute · review).

Hook (Claude Code)Lệnh tự chạy theo trigger (preToolUse, postToolUse, sessionStart...) — vd. pre-commit lint.
